# N3 — Feature

A **especificação detalhada** de uma única funcionalidade (ex.: Calcular Frete,
Cadastrar Cliente).

## O papel do N3

É o nível que a **equipe de desenvolvimento consome diretamente** para
implementar. Recebe um ID dentro do Feature Set (ex.: `USR-PRM-01` ou `F01`).

## O que contém

Descreve, campo a campo, o comportamento da funcionalidade:

- **Campos** — o que o usuário preenche e vê.
- **Regras de negócio** — as que valem só para esta feature.
- **Cenários** — em **Gherkin** (derivados dos critérios de aceite da história).
- **Comportamento de tela** — estados, validações e reações.
- **Origem** — a chave da história (ServiceNow) que originou a feature.
- **Implementação** — repositório e caminho do código.

## Artefato

```text
modules/[dominio]/[feature-set]/[feature].md
```

## Como gerar

| Passada | Prompt | Produz |
|---|---|---|
| **A — Negócio** | `PROMPT_3A_N3_negocio` | campos, regras e cenários em linguagem de negócio |
| **A (transcrição)** | `PROMPT_3A_N3_negocio_transcricao` | N3 a partir de uma transcrição de reunião |
| **B — Técnico** | `PROMPT_3B_N3_tecnico` | detalhe técnico e **dimensionamento (PF/CFP)** |

Para **alterar** uma feature existente, use `PROMPT_4A` / `PROMPT_4B`.

> Campos detalhados vivem em `global/DATA-MODEL.md` e requisitos não-funcionais em
> `global/NFR.md` — nunca dentro do N3.
