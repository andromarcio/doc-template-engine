# N1 — Domínio

As **grandes áreas funcionais** do sistema (ex.: Vendas, Suporte, Usuários).

## O papel do N1

É a **planta do sistema**: mostra como ele se divide em grandes áreas e como as
partes se relacionam, sem entrar em telas ou campos. Cada domínio recebe uma
**sigla de 3 letras** como ID (ex.: `USR`).

## O que contém

Em linguagem de negócio, cada domínio descreve:

- **O que faz** — sua responsabilidade central.
- **O que explicitamente não faz** — e a quem isso pertence.
- **Feature Sets** — os conjuntos de funcionalidades que o compõem (cada um com link para o seu N2).
- **Regras transversais** — válidas para todo o domínio.
- **Integrações** — o que o domínio lê de e escreve em outros domínios (leitura/escrita).

## Artefato

```text
modules/[dominio]/README.md
```

O índice consolidado de todos os domínios fica em `modules/INDEX.md`, atualizado
após cada N1 aprovado.

## Como gerar

| Passada | Prompt | Produz |
|---|---|---|
| **A — Negócio** | `PROMPT_1A_N1_negocio` | o domínio em linguagem de negócio |
| **B — Técnico** | `PROMPT_1B_N1_tecnico` | entidades, integrações e detalhe técnico |

> Caminho bottom-up: a partir de várias features (N3) é possível **derivar** o N1
> com `PROMPT_N3_TO_N1`.
