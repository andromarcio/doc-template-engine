# N0 — Visão de Produto

O nível **estratégico**. Define **por que o produto existe**: objetivo,
público-alvo e proposta de valor.

## O papel do N0

É o documento de referência mais alto — os demais níveis são **confrontados contra
ele** para garantir que não extrapolam o escopo nem contradizem os objetivos do
produto. O N0 **não detalha funcionalidades**; ele dá a direção que orienta todo o
resto.

## O que contém

- **Objetivo** — o problema que o produto resolve.
- **Público-alvo** — para quem ele existe.
- **Proposta de valor** — por que escolher este produto.
- **Limites de escopo** — em alto nível, o que está dentro e fora.

## Artefato

```text
global/N0_PRODUCT_VISION.md
```

## Como gerar

O N0 é o ponto de partida natural do fluxo **top-down**. A partir dele,
roda-se o `PROMPT_1A` para derivar os domínios (N1).

> Não há tabelas, telas ou endpoints no N0 — apenas a direção do produto.
